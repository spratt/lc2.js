Methods

lexer.lex(str, spec)
Given a string and a lexer specification, return matched tokens as an array. The expected format of the lexer specification is described below.

Specification

The lexer specification is an object with two properties, start_state, which is a string name of one of the states, which is an object whose values are arrays of pattern objects which are attempted to match against the string in order. Each pattern object must have the property regex which is matched against the input string, and may contain a next_state to transition to after a match, or a type whose value is a string and whose inclusion means when the regex matches to return a token of the matched characters with the specified token type.

Specification Object Properties

states
An object with user-defined keys whose values are arrays of pattern objects as described below. These pattern objects are attempted to be used to match the input string to the lexer in the order specified by the array.
start_state
A string matching one of the keys of states.

Pattern Object Properties

regex
A regular expression to be matched against the part of the lexer input string that hasn't yet been matched.
next_state (optional)
A string corresponding to the key of a state in the specification object described above. If the regex matches, transition to this state.
type (optional)
If the regex matches, return an object with two properties: type which is this value, and val which is the matched string.
